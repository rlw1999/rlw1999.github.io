Advanced Topics

Unions

Byte Ordering Revisited

  • Big Endian
    • Most significant byte has lowest address
    • Sparc
  • Little Endian
    • Least signficant byte has lowest address
    • Intel x86, ARM Android and IOS
  • Bi Endian
    • Can be configured either way
    • ARM

注意以不同的类型输出Union内的数据时要考虑大端小端

Memory Layout

  • Stack
    • 8MB limit
  • Heap
    • malloc()

      堆在管理动态内存时会倾向于将大数据和小数据分开，当然这与编译器有关，总的来说堆向上增长。

  • Data
    • Statically alloated as needed
  • Text/Shared Libraries
    • Read-only

      公用库的位置可变

Buffer Overflow

Avoid Overflow

  • avoid overflow vulnerabilities in Code
    • fgets instead of gets
    • strncpy instead of strpy
    • use %ns(like %4s)instead of integer
  • System-Level Protections
    • Randomized stack offsets
      • allocate random amount of space on stack
      • Shifts stack addresses for entire program
      • …
    • Nonexecutable code segments
      • add “executable” premission besides “read-only” and “writeable”
  • Stack Canaries can help
    • Check for corruption before exiting function
      • mov %fs:28, %rax

        %fs是段寄存器，后面的数字每次编译不同，放在缓冲区后面，用作哨兵，输入后检查哨兵是否被更改
