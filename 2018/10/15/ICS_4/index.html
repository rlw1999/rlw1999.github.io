Procedures

Mechanisms in Procedures

  • Passing control
  • Passing data
  • Memory management
    • Allocate during procedure excution
    • Deallocate upon return

x86-64 Stack

%rsp contains lowest element (top) of the stack

  • pushq Src
    • Fecth oprand at Src
    • Decrement %rsp by 8
    • Write operand ar address given by %rsp
  • popq Dest
    • Read value at address given by %rsp
    • Increment %rsp by 8
    • Store value at Dest
    • the memory doesn’t change

Passing Control

  • call label
    • Push return address on stack
    • Jump to label
  • ret
    • Pop address from stack
    • Jump to address

Passing data

  • First 6 arguments

    %rdi %rsi %rdx %rcx %r8 %r9

  • Return value
    %rax

Managing local data

  • Languages that support recursion
    • Code must be Reentrant
    • Need some place to store state of each instantiation
      • Arguments
      • Local variables
      • Return pointer
  • Stack allocated in Frames

Stack Frames

  • Contents
    • Return information
    • Local storage
    • Temporary space
  • Pointers
    • %rsp Stack Pointer
    • %rbp Frame Pointer(Optional, from IA32)

x86-64/Linux Stack Frame

  • Caller Stack Frame
    • Return address(push by call)
    • Arguments for this call
  • Current Stack Frame
    • “Argument build”(Parameters for function about to call)
    • Local variables(if can’t keep in registers)
    • Saved register context
    • Old frame pointer(optional)

      • 方便返回时找到caller的栈帧

callee可以访问并修改caller栈帧里的内容，比如caller向callee传递的参数大于6个时，就存在callee的栈帧里

Register Saving Conventions

  • Caller Saved
    • caller saves temporary values in its frame before the call

  • Callee Saved

    • Callee saves temporary values in its frame before using
    • Callee restores them before returning to caller

  • Caller Save

    • %rax
    • %rdi,…,%r9
    • %r10,%r11
  • Callee Save
    • %rbx, %r12, %13, %r14
    • %rbp
    • %rsp
