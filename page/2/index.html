有向图的强连通分支

任意节点之间相互可达，则称图是强连通的。图G的最大强连通子图称为G的强连通分支

Tarjan算法(有向图)

任取一点，用DFS的方法遍历整个图。用$dfs[i]$标记i节点的开始时间，$dfs[i]$越小表示节点越早。用$low[i]$表示从i出发DFS过程中能访问到最早的节点。初始时$low[i]=i$，但是如果在i出发的DFS中能够到达更早的节点j，则这个更早的节点j与i强连通，更新

维护DFS过程中的栈，当发现当前节点的子节点在栈中，则表明可以从该点出发走到更早的节点，更新$low[i]$。在遍历完所有子节点之后，用子节点的最小的$low[j]$更新父节点，即

当发现更新之后节点的$dfs[i]==low[i]$，则说明这个节点是当前强连通子图中最小的节点，于是将栈中所有i和i以上的节点出栈，构成一个强连通分量。

复杂度与深搜相同，为O(V)
算法正确性的证明需要把图中的节点按与u在栈中的关系分类，分别讨论与u在图中的关系。
感觉这个算法的关键是找到了栈与节点单调性之间的关系

无向连通图求割点和桥

删去割点后图不连通，去掉桥边后不连通

基本与Tarjan算法类似，注意在考虑节点i的子节点j是否在栈中时，$(i,j)$不能是之前深搜走过的边，也即j不能是i在深搜时的父节点

一个点$u$是割点，当且仅当满足下面两条

  • $u$是树根，$u$有多于一个子树
  • $u$不是树根，且存在$(u,v)$为树枝边(出现在DFS中的边,u为v的父节点)，且$dfs[u] \leqslant low[v]$

    因为此时v必须通过u与上面的节点连通，故去掉u图不连通

一条边$(u,v)$是桥，当且仅当

  • $(u,v)$为树枝边，且$dfs[u] < low[v]$

    注意这里是小于

  • 注意不能有重边

实际操作时，可以先做一遍DFS，再判断有无割点和桥

无向连通图点双联通分支

定义：不包含割点的极大连通子图
在求割点的同时可以顺便把每个点的双连通分支求出。在搜索时将边入栈，$(u,v)$满足$dfn(u)<=low(v)$或者u是树根时，u是双联通分支的根，将栈中$(u,v)$以上的边弹出组成一个双联通分支。
注意不要将反向边入栈两次，判断条件是

无向连通图边双连通分支

定义：不包含桥的极大连通子图
把桥边删除，原图就被分成了多个边双连通块，将双连通分支缩点，得到的图没有环，为一棵树。

在实际储存时，不需要开二维数组，直接遍历所有边更新$dist[\ ]$，此时在第k步求出的$dist[\ ]$中储存的值对应走法可能超过k条边(因为我们可能用更新过的u节点去更新v节点)。此时如果存在负权回路，用这种方法上面给的判断条件依然成立，依然可以判断出负权环；如果不存在负权环，那多走出环的路径一定更长，不影响最短路径的最终答案。

SPFA 算法

每次迭代，取出队头的点V，依次枚举从V能到达的U，用V更新U，如果U被更新，则将U的前驱记为V，如果U不在队列中，则将U入队，直到队列为空。如果一个点最短路被改进的次数达到n，则有负权环。

Trie图

多模式串的字符串匹配问题

  • 母串的当前字符，同时匹配多个模式串的当前字符
  • 失配时，母串指针不回溯，调整若干个模式串的指针到合适位置

Trie树(字典树，单词前缀树)

树中的每一个节点，都对应一个字符串，为从树根走到该节点的路径，根节点为空。

Trie图(AC自动机，DFA)

  • 对于插入的每个字符串，插入过程中使用的最后一个节点标记为终止节点
  • 如果只使用Trie树，在失配时只能回溯到根节点，并且母串指针回溯
  • 解决方法与KMP类似，对于每个节点，当在该节点失配时，应访问与其之前路径(已经与母串匹配)的后缀相同的最大前缀对应的节点，与母串当前字符比较。也就是树上的next[i]数组
  • 当按照上面描述的路径走到头时，应返回root，表明该次匹配不能匹配到任意模式串
  • 使用BFS构建前缀指针
    • 添加0号节点，所有子节点指向root
    • 访问到一个节点，其父节点与其连边对应字符ch，顺着其父节点的前缀指针一直往上走，直到访问到一个节点有对应ch的边，对应的子节点即为所求的前缀指针。

危险节点

  • 终止节点是危险节点
  • 如果一个节点的前缀指针指向危险节点，那么它也是危险节点。(对应一个模式串是一个另一个模式串的子串)
  • 在匹配过程中经过了危险节点，就表明匹配到一个模式串

K-th Number

You are working for Macrohard company in data structures department. After failing your previous task about key insertion you were asked to write a new data structure that would be able to return quickly k-th order statistics in the array segment.
That is, given an array a[1…n] of different integer numbers, your program must answer a series of questions Q(i, j, k) in the form: “What would be the k-th number in a[i…j] segment, if this segment was sorted?”
For example, consider the array a = (1, 5, 2, 6, 3, 7, 4). Let the question be Q(2, 5, 3). The segment a[2…5] is (5, 2, 6, 3). If we sort this segment, we get (2, 3, 5, 6), the third number is 5, and therefore the answer to the question is 5.
