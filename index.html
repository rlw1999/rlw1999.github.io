网络流问题

最大流

带权有向图$G=(V,E)$，有唯一的源和汇，并且对于任意其他 节点流入的流量等于流出的流量

一种思路：

  • 每次dfs寻找从源点到汇点有流量的路径，找到路径上容量最小的管道，再将路径上的管道容量减去此容量
  • 重复上一步，直到源到汇没有路径

但是这是有问题的，最终的最大流方法可以按上方法理解，但是不能用这种方法寻找，反例如下

改进：在每次dfs之后，在路径上增加反向边，容量设为搜索出的最小容量

  • 第一次

找到路径为a->b->c->d，容量为100
增加反向边

  • 增加反向边

这nm。。服气

总之就是叠加原理，用反向边标记当前管道可以减小的流量大小(叠加的流量不能与管道方向相反)。
每次增加的过程称为“增广”。

Dinic 快速网络流算法

使用BFS分层，在一次增广中找到多条路径。

  • 先使用BFS对残余网络进行分层，节点层数为从源点出发到节点所能走的最少边数
  • 分完层后，用DFS寻找增广路，要求DFS的每一步都必须走到下一层节点，搜索到与汇点同层的节点即结束搜索，若搜到的是汇点，则找到一条增广路径。
  • DFS找到一条增广路径之后，并不立即结束，而是回溯到某个节点继续寻找增广路径，回溯的节点u满足
    • DFS搜索树的树边(u,v)上的容量已经变为0，即刚刚找到的增广路径上增加的流量等于(u,v)增广前的容量
    • u是满足上条件的最上层节点
  • 如果回溯到源点而无法继续往下走，本次DFS结束
  • DFS结束后，再对残余网络再次进行分层，继续DFS
  • 当BFS无法到达汇点算法结束

Cache Memories

Cache memory organization and operation

垃圾mermaid，[()]都不能识别。。。

Direct Mapped Cache(E=1)

  • Cache
    • v(1 bit)+tag(t bits)+data block(2^b bytes)
  • Address
    • tag(t bits)+arr number(s bits)+b bits

每个数据块$2^b$bits，需要b个二进制位索引，实际内存有$2^{t+s}$个块，在cache中保留$2^s$个块，也即内存中$2^t$个块对应cache中的E个块
只有tag相同并且v=1时才命中
E=1时容易发生数据抖动

E-way Set Associative Cache

E路组相联Cache
当发生不命中时，将从内存中取出的block放在E个cache block中空的block内；当全满时，则根据不同的替换规则覆盖cache中的block。

General Cache Organization(S,E,B)

cache容量为$S\times E \times B$ bytes，$S=2^s$，$B=2^b$。

Writes

  • Write-through
    直接写进内存
  • Write-back
    写进cache，当被替换时写入内存，需要新增一位标记有没有被改过

What if Write-miss?

  • Write-allocate
    在cache中更新，被替换时写入内存
  • No-write-allocate
    直接写入内存

Typical

  • Write-through + No-write-allocate
  • Write-back + Write-allocate

Trie图

多模式串的字符串匹配问题

  • 母串的当前字符，同时匹配多个模式串的当前字符
  • 失配时，母串指针不回溯，调整若干个模式串的指针到合适位置

Trie树(字典树，单词前缀树)

树中的每一个节点，都对应一个字符串，为从树根走到该节点的路径，根节点为空。

Trie图(AC自动机，DFA)

  • 对于插入的每个字符串，插入过程中使用的最后一个节点标记为终止节点
  • 如果只使用Trie树，在失配时只能回溯到根节点，并且母串指针回溯
  • 解决方法与KMP类似，对于每个节点，当在该节点失配时，应访问与其之前路径(已经与母串匹配)的后缀相同的最大前缀对应的节点，与母串当前字符比较。也就是树上的next[i]数组
  • 当按照上面描述的路径走到头时，应返回root，表明该次匹配不能匹配到任意模式串
  • 使用BFS构建前缀指针
    • 添加0号节点，所有子节点指向root
    • 访问到一个节点，其父节点与其连边对应字符ch，顺着其父节点的前缀指针一直往上走，直到访问到一个节点有对应ch的边，对应的子节点即为所求的前缀指针。

危险节点

  • 终止节点是危险节点
  • 如果一个节点的前缀指针指向危险节点，那么它也是危险节点。(对应一个模式串是一个另一个模式串的子串)
  • 在匹配过程中经过了危险节点，就表明匹配到一个模式串
