UVa 1471 Defense Lines

After some hard negotiations, it appeared that building new towers is out of question. Mages of
Ardenia have agreed to demolish some of their towers, though. You may demolish arbitrary number of
towers, but the mages enforced one condition: these towers have to be consecutive.
For example, if the heights of towers were, respectively, 5, 3, 4, 9, 2, 8, 6, 7, 1, then by demolishing
towers of heights 9, 2, and 8, the longest increasing sequence of consecutive towers is 3, 4, 6, 7.

基本的计算方法是先预处理计算G[i]和L[i]，表示以第i元素结尾的最长连续上升序列和以i元素开头的最长连续上升序列，然后再搜索L[i]+G[j](A[i]>A[j])的最大值。
问题是直接搜索复杂度应该在n^2,maxn=200000,肯定会超时。枚举区间头或尾，另一头的搜索必须控制在logn,此时就需要构造单调序列，使用二分查找。
枚举区间尾i,搜索区间头j。在搜索区间头时，需要考虑序列值A[j]和G[j]，找到满足A[j]<A[i]的最大的G[j]。可以将A[j],G[j]合并为二元组，按照A[j]的大小进行排序，对于一种情况需要注意，当A[j]<=A[j'],g[j]>=g[j'],应该直接扔掉j’，(因为一定不会选择j’作为开头) 这样g[j]也是按照从小到大排序。此时可选择STL中的set作为容器，方便执行insert(),erase(),lower_bound()。

另外一种维护单调序列的方法是保存每个G[j]对应的最小的A[j]，这种方法经常在寻找子序列中出现。

UVa 1606 Amphiphilic Carbon Molecules

Posted on 2018-07-21 | In Algorithms

UVa 1606 Amphiphilic Carbon Molecules

平面上有黑点和白点，要求找到一条直线，使得直线一侧的黑点加另一侧的白点个数最大，在直线上的点可以算在任意一边。复杂度应该限制在n^2logn。
这道题的解法相当精妙。首先考虑一般情况，可以发现直线一定可以经过至少两个点，于是可以通过枚举通过直线的两点，计算两边的点个数，但是直接枚举复杂度应该在n^3。
为了降低复杂度，可以固定一个点，然后将其他点按与该点形成的向量的辐角排序，直线依次扫过这些点，就可以动态地维护两侧点个数。
具体实现时，一些细节可以大大简化代码。

  • 使用atan2()函数可以稳定地计算辐角，返回值在$(\pi,\pi]$之间,注意的是参数顺序是(y,x);
  • 在统计白点黑点个数时，可以先将所有黑点按基点中心对称，这样只需要统计一侧的点个数
  • 将直线旋转360度，从点0到点N-2，即可历遍直线的两侧

how many quadruplet (a, b, c, d) ∈ A × B × C × D are such that a + b + c + d = 0. In the following, we
assume that all lists have the same size n.

n<=4000,9000ms,因此算法的复杂度应该控制在n^2logn,有两种方法

Hash Table

容易想到的方法是将A+B的和记录下来，然后枚举C+D的和，查找记录表中的统计结果然后加起来。但是我们不可能将A+B的和放在一个 2^32 的数组里，这里就用到了哈希表的方法
将所用可能和的情况映射到一个1e7大小的一个Hash Table中(理论上来讲开到maxn^2可以完美解决冲突问题，数据结果是1e7也能过,但是1e6就TLE了)，然后在Hash Table中查找
这种方法的复杂度应该在$n^2$和$n^2logn$之间

Binary Search

虽然整数的范围太大，但是maxn^2的数据量还是可以接受的，可以把A+B放进map里，查找速度为logn,但是如果STL中的map，常数太大，结果是超时，这里需要自己排序然后二分查找

For example, in order to make {2, 4, 1, 5, 3, 6}, you can cut 1 and paste before 2, then cut 3 and
paste before 4. As another example, one copy and paste is enough for {3, 4, 5, 1, 2}. There are two
